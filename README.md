# eiyou291

管理栄養士国家試験（午前）対策の **291問 問題集アプリ**。
ビルド不要・ブラウザで開くだけで使えます（v39 から CSS/JS を同階層の `app.css`・`app.js` に分離、v41 からルート `index.html` がアプリ本体）。スマートフォン・iPad にも対応し、ホーム画面に追加すればアプリのように使えます。

## 特徴

- **午後問題の追加を開始（v53〜・データ駆動）** — 午前に加え、**午後**を順次収録中。午後は `pm.js`（公式問題・正答に基づくデータ＋教科書準拠の解説）に書き、`app.js` の `renderPM()` が既存と同じ `article.q` 構造に描画するため、検索・絞り込み・○✕マップ・学習トラッカー・模試がそのまま使えます。科目フィルタ・弱点分析に**午後科目**（栄養教育論／臨床栄養学／公衆栄養学／給食経営管理論／応用力試験）を追加（問題がある科目だけ表示）。まず第39回 午後「栄養教育論」からパイロット投入。
- **午前291問を収録** — 各問に選択肢・正誤判定・詳解・直前暗記メモ・ひっかけ対策付き。
- **表（資料）問題の表組み化（v48〜v51）** — 【資料】の数値が縦に羅列されて読めなかった表問題を、正しいHTML表として描画（横スクロール対応）。**第38〜40回すべての表問題（計17表）を厚労省公式PDF（ユーザー提供）と照合して正確に再構築**。元データがPDF抽出で欠落・崩壊していた表（喫煙コホート q40-5／成分表の記号A〜I q40-66／スクリーニング2×2 q39-5／年齢調整死亡率 q39-4／なすエネルギー q39-67／身体活動 q39-93／身長体重 q38-92 ほか）も、公式の数値で補完済み。数値はすべて公式照合済みで、表として読みやすく表示。トマトジュースの栄養成分表示（q38-57）も表として再現し、**18の資料すべてが読みやすい表で表示**される。
- **ファーストビューの整理（v47）** — 問題の前に大きなカードが7枚並んでいたのを整理。「正答チェック・使い方／ひっかけ実例集／直前暗記表／図解で覚える」の4つを **1つの折りたたみ「📚 参考・暗記・図解ツール」に集約**（既定は閉じた状態）。目立ちすぎていた「全体クリア」の大きな黄色バーは、控えめな小さなボタン（「↺ 答えの表示をまとめて消す／記録は残ります」）に縮小。結果、トラッカーの下がすっきりし、問題まで早く到達できる。
- **弱点克服の連動（v46）** — 「科目別の弱点分析」を⚙ツールの奥からトラッカー直下に移動して常時表示。各科目バーを**タップするとその科目の「間違いだけ」を即復習**（未挑戦の科目は自動で「未挑戦」表示にフォールバック、全問正解なら全問見直し）。間違いがある科目は赤くハイライトし「🔁 弱点 N問を復習 →」と件数つきの導線を表示。出題回・難易度フィルタは自動で全解除して科目に集中。
- **範囲表示で共有・version.json（v45）** — URL に `?range=40-85` を付けると、出題順の通し番号（1〜291、シャッフルに依らず固定）で 40〜85問目だけを表示。学習範囲を絞ってリンクで共有でき、上部の「全問に戻す」で解除（状態は `S.range` として検索・科目・復習・模試フィルタと合成）。範囲外・不正値は無視。単一問題への共有は従来どおり `#q40-85`（第40回問85）でアンカースクロール。公開確認用 `version.json`（SWキャッシュ対象外で常に最新）を追加し、「端末キャッシュか配信ズレか」を一発で切り分け可能に。回帰テストに範囲表示2項目を追加。
- **シャッフル強化・本番モード（v44）** — 検索欄の下に常時表示の「🔀 問題順」「🔀 選択肢」トグルを追加（⚙ツールに埋もれていた機能を前面化）。**選択肢シャッフル**は各問の選択肢の並び順だけを入れ替え（位置で答えを覚えるのを防ぐ。選択肢番号や解説テキストは原文のままで正誤判定・○✕マップ・解説の対応は崩れない）。シャッフル設定（問題順・選択肢）は localStorage に保存し、リロード・アプリ再起動後も維持。「🎯 本番モード」は問題順＋選択肢を両方シャッフルし、全問の模試をワンタップで開始。回帰テストに4項目追加（計21項目すべて合格）。
- **更新耐性・診断機能（v43）** — 表示が古いまま更新されない端末向けに「🔄 最新版を取り直す」ボタンを⚙ツールに追加（Service Worker とキャッシュを破棄して再取得。**学習記録（localStorage）は消さない**）。Service Worker 登録を `updateViaCache:'none'` にし、SW スクリプト自体が古いキャッシュに固定される問題を防止（`reg.update()` の失敗も握りつぶしてコンソールを汚さない）。`?debug=1` を付けて開くと、画面に `app=v43 / q=291 / sw=on / cache=…` の診断オーバーレイを表示。ヘルプにも古い版から抜け出す手順を追記。
- **キャッシュ堅牢化・模試採点の改善（v42）** — `app.css`/`app.js` に `?v=` を付与しブラウザ／iOSの古いキャッシュ掴みを防止。`.nojekyll` を追加し GitHub Pages を素のまま配信。Service Worker を起動時＋1時間ごとに `reg.update()` で更新確認。模試の**未回答を不正解として復習対象に**記録（既に正解済みの実績は壊さない）。`console.info([EIYOU291] …)` で版数を確認可能。版数は `app.js` の `APP_VERSION`／`sw.js` のキャッシュ名／`index.html` の `?v=` の3点で管理。
- **公開導線の整理・バグ修正（v41）** — ルート `index.html` をアプリ本体に統一し、版ごとのHTMLファイル名を廃止（入口が旧版を指す事故を構造的に排除。版数は `app.js` の `APP_VERSION` と SW キャッシュ名のみで管理）。模試開始・進捗リセット時に内部選択状態 `a._sel` をクリア（「同じ選択肢が解除扱いになる」不具合を修正）。待機中の Service Worker でも「🆕更新」バーを表示、SW登録条件を `isSecureContext` に。
- **状態管理を1か所に集約（v40）** — 進捗・★・続きから・表示モード・集中・模試などの状態を単一の `S` オブジェクトに統合（散在していた変数を整理し、機能間の干渉を追いやすく）。localStorage のキーは従来どおりで既存データは保持。回帰テスト15項目すべて合格を確認。
- **保守性・堅牢性の改善（v39）** — CSS/JS を `app.css`・`app.js` に分離（巨大HTMLから切り離し、レビュー・デバッグしやすく）。Service Worker を堅牢化（インストール時のキャッシュ失敗を握りつぶさない／ページ遷移のみHTMLフォールバック、CSS・JS・画像は失敗時にHTMLを返さない／キャッシュ名を日付付きに）。エラーログと localStorage の安全ラッパー（`safeLoad/safeSave`）を追加。回帰テスト（`tests/regression.mjs`）と確認手順（`tests/README.md`）を整備。
- **文字・余白を見やすく（v37→v38）** — 余白・行間・選択肢番号の見やすさを調整。v38で文字サイズは標準的な大きさ（問題文15px・選択肢14px）に戻し、大きすぎないバランスに（さらに変えたい場合は⚙ツールの A−／A＋ で4段階調整可）。
- **問題文の読みやすさ改善（v35→v36）** — 元データ由来で文の途中に入っていた不自然な改行を整形。v36では「文末（。）や表・資料の区切りは残し、文の途中の改行だけをつなぐ」賢い処理に変更し、表を含む問題（資料つき）でも崩れないようにした（全291問対応）。
- **選択肢のトグル開閉（v34）** — 選んだ選択肢をもう一度タップすると、解説を閉じて未選択の状態に戻る（記録も取り消し）。問題の頭へ戻ってすっきり解き直せる。
- **問題の区切りを明確化（v33）** — 各問カードに色付きの左ライン（未挑戦＝出題回の色／正解＝緑／不正解＝赤）・余白拡大・影・見出し下の区切り線を追加。スクロール中も1問ずつの境目と解答状況が一目で分かる。
- **品質改善・ヘルプ追加（v32）** — 医療内容のファクトチェック実施（図解10点はすべて正確。食中毒のO157に「生体内毒素型」の注記を追加）。テスト（模試）中に正誤の色が見えてしまう不具合や、採点後のキー操作で進捗が書き換わる不具合など複数のバグを修正。冒頭に「❓ はじめての方へ（使い方・色の意味）」を追加し、緑＝正解／赤＝不正解／青＝選択中・操作／黄＝注意 の**色の凡例**を明示。
- **操作パネルのコンパクト化（v31）** — 学習トラッカーを大幅にスリム化（高さ約半分）。常時表示は「正答率・正解/不正解/残り・続きから・検索・表示モード」だけにし、テスト/集中/文字サイズ/進捗データ等は「🧭 学習メニュー・ツール」に集約。出題範囲（回・難易度・科目）は折りたたみ化。画面下の固定チップを廃止し、指標表記も簡潔化。**出題回/難易度/科目フィルタが効かなかった既存不具合も修正**。
- **テスト（模試）モード（v30）** — 現在の表示対象からランダムに 10／20／30／全問を出題。解答中は答えを隠し、「採点する」で得点・正答率を表示。終了後は「間違いだけ復習」へ。結果は学習トラッカーにも反映。
- **解答後に解説を自動表示＋「次の問題へ」（v30）** — 選択肢を押すと解説が自動で開き、各問に「次の問題へ →」ボタンを表示（⚙ツールでオン/オフ）。
- **キーボード操作（v30）** — `1`〜`5` で現在の問題に解答、`←`／`→` で前後の問題へ移動（iPad＋キーボード／PC）。
- **1問ずつ集中モード（v29）** — 長いスクロールをやめ、1問だけ大きく表示→「次へ／前へ」で進む。現在のフィルタ・表示モードに連動。スマホ・スキマ時間の集中学習に。
- **★ブックマーク（v29）** — 気になる問題に★を付け、「★だけ」表示で見直し。不正解とは別管理。
- **続きから再開（v29）** — 最後に解答した／集中モードで見ていた問題へワンタップで戻る（再読込・端末再起動後も保持）。
- **進捗のバックアップ・復元（v29）** — 進捗・★・設定を JSON で書き出し／読み込み。端末を変えても引き継げ、配った相手にも初期データを渡せる。
- **画面構成・配色の刷新（v28）** — ファーストビューを短縮（導入文を整理し、トラッカー直下にすぐ問題が来るレイアウト）。二次的な操作（出題・文字サイズ・科目別）は「⚙ ツール」に集約。配色は cyan=操作／green=進捗・正解／amber・red=注意 に役割を整理し、コントラストを向上。科目別バーは未挑戦をグレーで区別。
- **図解で覚える（v27）** — 高頻出テーマを教科書準拠の図解で整理（食事摂取基準の5指標カーブ、感度・特異度の2×2、ホルモンの作用方向、アレルギーⅠ〜Ⅳ型、周産期・新生児・乳児死亡の区分、疫学の2×2と式、社会保障の4本柱、ビタミン、食中毒、WBGTの運動指針 ほか全10点）。
- **選択肢○✕マップ（v27）** — 各問に、選択肢ごとの ○／✕ と「正しくは」を一覧化した整理ビューを自動生成。「📊 選択肢マップ」で全問を一括開閉。
- **科目別の弱点分析（v27）** — 学習トラッカーに科目別の正答率バーを表示。苦手分野（正答率が低い科目）が一目で分かる。
- **文字サイズ切替（v27）** — A−／A＋で問題文の表示倍率を4段階に変更（設定はブラウザに保存）。
- **ランダム出題・シャッフル（v27）** — 表示順をランダム化／元の順に戻す。本番の順不同に慣れる用。
- **印刷・PDF用レイアウト（v27）** — 操作UIを隠し、問題＋選択肢＋○✕マップを白背景で整形印刷（紙で配る・PDF保存用）。
- **学習トラッカー（v26）** — 解答すると正解／不正解を自動記録し、**正答率・正解数・不正解数・未挑戦数・最大連続正解**をダッシュボード表示。記録はブラウザ（localStorage）に保存され、次回起動時に**選んだ選択肢まで復元**します。
- **復習モード（v26）** — 「不正解だけ」「未挑戦だけ」に絞って表示。間違えた問題の解き直し・残りの消化がワンタップ。
- **キーワード検索（v26）** — 設問・テーマ・選択肢の文言から問題を即時フィルタ（例：インスリン／周産期死亡率／WBGT）。
- **iPad / iPhone アプリ化（v26 / PWA）** — `manifest.webmanifest`・Service Worker・アイコンを同梱。Safari の「ホーム画面に追加」で全画面起動、オフラインでも動作。新バージョンは「🆕更新」バーで通知。
- **3軸フィルタ** — 出題回・難易度・科目で絞り込み可能（検索・復習モードと併用可）。
- **学習フロー** — 「問題を読む → 選択肢を押す → 解説を見る」のシンプルな導線。
- **モバイル最適化** — ダークテーマ、固定ヘッダ、`viewport-fit=cover` 対応。
- **JS無効でも動く** — フィルタや解説の開閉は CSS（ラジオ/チェックボックス）で実装。トラッカー・検索・復習・PWA のみ JavaScript で追加しており、JS が無効でも問題集としてそのまま使えます。

## 使い方

1. リポジトリをクローン、または HTML ファイルを直接ダウンロードします。
2. ルートURL（公開時）またはローカルなら `index.html`（`app.css`/`app.js` と同じ階層）をブラウザで開きます。
3. 画面上部のフィルタ・検索・表示モードで出題範囲を絞り込みます。
4. 問題を読み、選択肢を押すと正誤と詳しい解説が表示され、学習トラッカーに記録されます。

> サーバーやビルドは不要です。ファイルをダブルクリックするだけで動作します。

### iPad / iPhone でアプリとして使う（PWA）

進捗の保存やオフライン動作・ホーム画面追加を完全に活かすには、`file://` で直接開くのではなく **HTTP(S) で配信**してください（Service Worker は `https`/`localhost` でのみ動作）。

- **手軽に試す（ローカル配信）**: リポジトリのルートで `python3 -m http.server` を実行し、`http://<PCのIP>:8000/` を iPad の Safari で開く。
- **公開する**: GitHub Pages 等にこのリポジトリを配信し、ルートURL（`index.html`）を開く。
- **ホーム画面に追加**: Safari の共有メニュー →「ホーム画面に追加」。全画面（standalone）で起動し、オフラインでも動作します。新バージョンを配信すると「🆕更新」バーが出るので、押すと最新化されます。
